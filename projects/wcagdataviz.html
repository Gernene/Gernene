Web Accessibility Checklist

1.1.1: Non-decorative non-text content has a text alternative. (A)
1.2.1: Alternatives are provided for pre-recorded video-only and audio-only media. (A)
1.2.2: Captions are provided for pre-recorded audio. (A)
1.2.3: Audio descriptions (narration of important visual details) or alternatives are provided for pre-recorded media. (A)
1.2.4: Captions are provided for live audio content. (AA)
1.2.5: Audio descriptions are provided for all pre-recorded video. (AA)
1.3.1: Information, structure, and relationships can be determined programatically or through text. (A)
1.3.2: Correct reading sequence can be programatically determined. (A)
1.3.3: Instructions for operating content don't rely soley on sensory characteristics like shape, color, position, or sound. (A)
1.4.1: Color is not the only means of conveying information. (A)
1.4.2: Automatic audio that plays for more than 3 seconds has a pause, stop, or volume control mechanism. (A)
1.4.3: Non-decorative text has a constrast ratio of at least 4.5:1. Large text has a contrast ratio of at least 3:1. (AA)
1.4.4: Text (excluding captions and images) can be resized up to 200%. (AA)
1.4.5: Text is used instead of images of text unless image presentation is essential or can be customized. (AA)
2.1.1: Functionality is operable via keyboard unless input depends on the path of movement. (A)
2.1.2: If focus can be moved to a component via keyboard, focus can be moved away via keyboard. (A)
2.2.1: Content time limits are removable or adjustable. (Exceptions exist) (A)
2.2.2: Users can pause, stop, or hide blinking, scrolling, or auto-updating information. (A)
3.1.1: The default language of a page can be programatically determined. (A)
3.1.2: The language of each individual passage or phrase can be programatically determined. (AA)
3.2.1: When a component receives focus, it does not initiate a change in context. (A)
3.2.2: Changing component settings won't automatically cause a change in context unless the user has been advised. (A)
3.2.3: Navigation/navigation order is consistent across a website. (AA)
3.2.4: Components with the same functionality are identified consistently. (AA)
3.3.1: Input errors are described to the user in text. (A)
3.3.2: Labels or instructions are provided when content needs input. (A)
3.3.3: Suggestions are provided for input errors. (AA)
3.3.4: Actions involving legal commitments, finances, and data are reversible, checked, or can be confirmed by the user. (AA)
4.1.1: Markup elements are properly nested, don't contain duplicate attributes, and have unique IDs. (A)
4.1.2: The name and role of user interface components can be programatically determined. (A)
